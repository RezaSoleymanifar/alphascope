# Alpha Archive methodology

Every paper goes through the same pipeline. No cherry-picking, no parameter tuning per paper. This document is the public methodology spec.

## 1. Universe

Default: SP500 constituents (point-in-time membership). Override per paper if author specifies (e.g., Russell 1000, Russell 3000).

We use point-in-time membership where available — no survivor bias.

## 2. Data alignment

- **Prices**: split + dividend adjusted, daily close
- **Fundamentals**: lagged by 1 quarter (45-day reporting delay assumption) to prevent lookahead
- **Macro**: FRED series, no revision (use vintage data where available)
- **Holidays / corporate actions**: handled by data vendor

## 3. Feature computation

Generated by LLM from paper spec. Human-reviewable before execution. Gated by sandbox:
- No network access
- No filesystem writes outside designated tmp
- Time limit per execution
- Memory limit per execution

## 4. Cross-validation

**Purged k-fold with embargo** (Lopez de Prado 2018):
- Split data into N=10 chronological groups
- Test on each group; train on remaining
- **Purge**: drop training samples whose labels overlap test
- **Embargo**: drop training samples in 1% of dataset immediately after test

For each paper, we also produce:
- **Single walk-forward backtest** (sanity check)
- **CPCV** (Combinatorial Purged CV, k=2) for Sharpe distribution

## 5. Performance metrics

For each backtest result we report:

### Information Coefficient (IC)
```
IC_t = Spearman_corr(predicted_t, realized_t+horizon)
```
- `ic_mean`, `ic_std`, `icir = mean(IC) / std(IC) × √N`
- `ic_t_stat`, `ic_p_value`

### Strategy stats
- `sharpe`, `sharpe_oos` (annualized)
- `deflated_sharpe` (DSR-corrected, see §6)
- `ann_return`, `ann_vol`, `max_drawdown`, `calmar`
- `turnover`

### Replication score
- `claimed_sharpe / measured_sharpe`
- 1.0 = perfect replication; <0.5 = weak; >2.0 = author over-claimed

## 6. Multiple-testing correction (Deflated Sharpe Ratio)

Bailey + Lopez de Prado (2014):
```
DSR = Φ((SR - E[max SR | null]) × √(T-1) /
        √(1 - γ_3·SR + ((γ_4-1)/4)·SR²))
```

We correct for:
- N = total signals tested across the Alpha Archive platform (cumulative)
- Skew + kurtosis of strategy returns
- Sample size T

DSR threshold for "ship": > 0.95 (analogous to t-stat > 2 with multiple-testing).

## 7. Cost model

Standard for all papers (no per-paper tuning):
- **Round-trip cost**: 5 bps (commission + slippage baseline)
- **Market impact**: σ × √(Q/ADV) where Q = order size, ADV = avg daily $ volume
- **Borrow cost** (for shorts): 30 bps annualized default
- **Financing**: ignored at MVP (assume cash equity, no leverage)

Total cost applied per rebalance via turnover × cost_bps + impact.

## 8. Regime analysis

Each backtest reports performance broken down by:
- VIX quintile (low to high)
- NBER recession indicator
- Rate regime (Fed funds vs neutral)
- Pre/post 2020 (regime change marker)

A signal that only works in 1 regime is flagged, not killed.

## 9. Out-of-sample window

Every paper:
- **Training window**: last 60 months before OOS
- **OOS window**: last 24 months, frozen — never used for parameter selection
- **OOS-vs-IS Sharpe ratio**: reported as `oos_decay`. <50% = significant decay.

## 10. Verdict criteria

| Verdict | Conditions |
|---|---|
| **ship** | DSR > 0.95 AND OOS Sharpe ≥ 0.5 × IS Sharpe AND ICIR > 0.3 AND survives ≥7/11 sectors |
| **iterate** | Some criteria met but not all |
| **kill** | DSR < 0.5 OR OOS Sharpe < 0 OR ICIR < 0.1 |

Verdicts are public and re-computed each time the backtest is rerun.

## 11. Reproducibility guarantees

For every published result:
- Original paper link
- LLM-extracted spec (versioned)
- Generated Python code (versioned, public)
- Data vintage (price source + date)
- Parameters used (cost bps, train window, etc.)
- Seed for stochastic components
- Backtest run timestamp

A user can `git clone alpha-archive && uv run alpha-archive replay <result_id>` and reproduce the result byte-for-byte.

## 12. Honest limitations

- **MVP data quality**: yfinance is good enough for most factor papers but not for HFT or microstructure work. Future: switch to vendor data.
- **Universe drift**: SP500 PIT membership is approximated until full Compustat / CRSP integration.
- **Survivorship**: best-effort; depends on vendor data quality.
- **Lookahead**: LLM-generated code is reviewed; can still introduce subtle leaks.
- **Generalization**: Alpha Archive replicates **published methodology**. Novel discoveries by authors not in the paper are not captured.
